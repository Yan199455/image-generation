# 输入规范

定义 hero-banner-bg Skill 的所有输入字段、约束、默认值。

## 输出形态（系统固定，用户不可选）

每次生成产出**固定 2 套图，共 4 张**：

```
Set 1：
  ├── 桌面图 1920×600（Variant A）
  └── 移动图 1080×1080（Variant A，主体一致 + 构图映射）

Set 2：
  ├── 桌面图 1920×600（Variant B）
  └── 移动图 1080×1080（Variant B，主体一致 + 构图映射）
```

**为什么固定**：

- 真实产品上线必须双端配套，桌面+移动同步交付才能直接用
- 2 套（4 张）是最小可决策样本量：少了用户没得选，多了浪费 token
- 同一套的桌面和移动用同一个 Scene Variant，保证主体和场景一致

**系统固定参数（不出现在用户输入里）**：

| 参数 | 固定值 | 说明 |
|------|--------|------|
| 输出端 | 桌面 + 移动 | 始终双端 |
| 桌面尺寸 | 1920×600 | 横向 16:5，标准 PC Hero |
| 移动尺寸 | 1080×1080 | 1:1 方形，移动端通用 |
| 套数 | 2 | 每次固定 2 套 |
| 总图数 | 4 张 | 2 套 × 2 端 |
| 模型调用次数 | 4 次 | 每张图独立调用 GPT-Image-2 |

> 这些参数在未来 v1.1 可能开放为可选项（如增加 1080×1920 全屏移动尺寸、3 套以上等）。当前为简化决策强制固定。

## 必填字段（2 项）

| 字段 | 取值范围 | 备注 |
|------|----------|------|
| 行业 | `_shared/references/enums.md` 中"行业枚举"的 14 个之一 | 模糊匹配用户输入到对应枚举 |
| 文案区域位置 | 左留白 / 右留白 / 居中下方留白 / 通屏氛围 | 桌面构图依据；移动版会自动映射（见 mode-spec.md） |

**校验规则**：

- 必填项缺失 → 直接向用户追问，不要假设
- 行业匹配模糊：用户说"我做驾考的" → 应映射到"教育 / 驾考"
- 行业匹配失败 → 列出所有 14 个枚举供用户选择

## 选填字段（含默认值）

| 字段 | 取值范围 | 默认值 | 默认值来源 |
|------|----------|--------|-----------|
| 视觉表现形式 | 该行业的 4 个预设之一 | 该行业的★默认预设 | `_shared/references/industries/<slug>.md` 中的 `default_preset` |
| 情感表达 | 6 个之一（专业可信 / 朴实可信 / 温暖治愈 / 简洁清爽 / 活力激发 / 神秘梦幻） | 该行业的默认情感 | `_shared/references/industries/<slug>.md` 中的 `default_emotion` |
| 关键词 | 自由输入 | 空 | — |
| 输出格式 | WebP / PNG / JPG | WebP | — |
| 排除元素 | 自由输入 | 空（系统仍自动追加通用负向词 + 联动负向词） | — |

> 已移除字段：**图片尺寸**、**生成数量**。这些现在由系统固定。

## 关键词字段约束

- **最多 3 个**。超出时只取前 3，向用户提示
- 用逗号 / 顿号 / 空格分隔均可
- 关键词类型不限：产品场景词、视觉具象词、概念词
- 关键词进入两条流水线：
  1. 加到正向 Prompt 的 "Additional content focus"
  2. 触发"关键词语义 → 自动负向词联动"（见 `_shared/references/negative-rules.md`）

## 输入解析示例

### 完整输入

```
行业: 教育 / 驾考
视觉表现形式: 真人学习场景
情感表达: 温暖治愈
关键词: 驾考, 学车, 高通过率
文案区域位置: 左留白
输出格式: WebP
排除元素: 紧张氛围, 严肃考场
```

### 极简输入（只填必填）

```
行业: 住房 / 房地产
文案区域位置: 左留白
```

→ 其余字段全部走默认值。

→ 系统自动产出：1920×600 桌面图 ×2 + 1080×1080 移动图 ×2（共 4 张，2 套）。

### 自然语言输入（需要 Skill 自己解析）

```
帮我生成一张驾考网站的 Hero Banner，文案放左边
```

解析为：

```
行业: 教育 / 驾考
文案区域位置: 左留白
（其余走默认值）
```

→ 同样产出 2 套 4 张图，无需用户指定数量或尺寸。

## 输入解析的优先级

1. 如果用户用结构化字段输入，直接解析
2. 如果用户用自然语言，提取关键信息，缺失项用默认值
3. **必填项无法识别**时，向用户追问；不要默认填一个值
4. 用户若试图指定"生成 8 张" / "只要桌面" / "1080×1920 移动版" → 友好告知当前 v1.0 固定为 2 套 + 1920×600 + 1080×1080，未来版本会开放
