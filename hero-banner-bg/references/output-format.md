# 用户回复格式

定义 hero-banner-bg Skill 给用户的标准回复模板。

## 标准回复模板（有图模式）⭐ 双端配对

每次输出 **2 套 = 4 张图**，按"套"分组展示，每套内桌面 + 移动并排或上下排列。

```
【输入解析】
行业: <值>
文案位置: <值>（桌面构图）→ 移动构图: <自动映射后的值>
视觉表现形式: <值>（默认 / 用户指定）
Quality Module: <Mx 编号>
情感表达: <值>（默认 / 用户指定）
关键词: <值 或 无>
输出形态: 2 套 × 双端 = 4 张（系统固定）
图片格式: <WebP / PNG / JPG>

【变体分配】
Set 1 → Variant A: <variant_id> | <中文 tag>
Set 2 → Variant B: <variant_id> | <中文 tag>

【合成 Prompt（Variant A 示例）】
Positive (桌面): <英文正向 prompt 全文>
Positive (移动): <英文正向 prompt 全文，仅 Block 7 不同>
Negative: <共享负向 prompt 全文>

【生成结果】

▼ Set 1（Variant A：<中文 tag>）
桌面 1920×600  | 移动 1080×1080
[嵌入桌面图]   | [嵌入移动图]

▼ Set 2（Variant B：<中文 tag>）
桌面 1920×600  | 移动 1080×1080
[嵌入桌面图]   | [嵌入移动图]

【操作提示】
- 想换风格？回复"换视觉表现形式: XXX"
- 想调情绪？回复"换情感: XXX"
- 想加约束？回复"加排除: XXX"
- 想锁定某套主体？回复"锁定 Set N，重生成另一套"
- 想重新生成？回复"重新生成"
```

通用调用参数：

## 输入不完整的回复模板

当**必填项缺失**，不要假设、不要默认填充，直接追问：

```
为了生成 Hero Banner 背景图，我还需要确认以下信息：

【必填】
- 行业：你的产品属于哪个行业？可选：
  住房 / 房地产、手机/通信/补贴、金融/银行/信用卡/税务、医疗/医院/牙科、
  法律、教育/驾考、本地服务/政府/目录、餐饮/优惠券/消费、汽车、宠物、
  App/工具下载、物流追踪、模板文档、占星娱乐
- 文案区域位置：你打算把首屏文字放在画面哪一侧？
  左留白 / 右留白 / 居中下方留白 / 通屏氛围
  （注：移动版会自动映射到合理位置，你不用单独考虑移动构图）

回复这两项后我就开始生成。其他配置（情感、关键词等）我会用默认值，你也可以一起说。

提示：每次生成会出 2 套 × 桌面+移动 = 共 4 张图，尺寸固定为 1920×600 + 1080×1080。
```

## 行业未完整的提示模板

当用户选择的行业**还未完整填充**（reference 文件中关键内容是 `[待补充]`）：

```
注意：「<行业名>」行业目前 Skill 还未完整填充（v1.0-skeleton 阶段）。

我可以：
A. 用兜底方式继续生成 → 用 Quality Module 的通用质感 + 一段泛化的预设描述，能出图，但风格泛化、变体单一
B. 先用「住房 / 房地产」行业测试 → 这是目前唯一完整填充的行业，可以验证整套流程

你选哪个？或者直接告诉我"用 A 兜底跑"，我就继续。
```

## 用户试图修改固定参数时的提示

```
你想 <修改尺寸 / 修改数量 / 只要桌面 / 只要移动>。当前 v1.0 这些参数是固定的：
- 输出形态：固定 2 套 × 桌面+移动 = 4 张
- 桌面尺寸：1920×600
- 移动尺寸：1080×1080

这是为了简化决策、保证桌面移动主体一致。
未来 v1.1 会开放为可选项。当前如果继续，我就按默认配置生成。继续吗？
```

## 错误处理回复模板

### 关键词超过 3 个

```
你给了 <N> 个关键词，本 Skill 上限是 3 个（避免 Prompt 失焦）。
我只取前 3 个："<k1>", "<k2>", "<k3>"
其余的 <剩余> 个被忽略。如果你觉得"<被忽略的>"很关键，告诉我替换哪一个。
```

### 行业匹配失败

```
我没识别出"<用户输入>"对应哪个行业。本 Skill 支持以下 14 个：
[列出全部]
你的产品最贴近哪个？
```

## 多轮对话的状态保持

用户在拿到 2 套结果后常见的后续指令，按以下逻辑处理（不重新解析全部输入，只覆盖变化的字段）：

| 用户指令 | 处理方式 |
|---------|---------|
| "换视觉表现形式: XXX" | 替换 visual_preset，重新选 2 个 Variant，重新执行 Step 4-10 |
| "换情感: XXX" | 替换 emotion，重新执行 Step 5-10 |
| "加排除: XXX" | 追加到 user_excluded_elements，重新执行 Step 6-10 |
| "锁定 Set N，重生成另一套" | 保留 Set N 的 Variant，另一 Set 选不同 Variant 重新生成 |
| "重新生成" | 保持所有输入，重新随机选 2 个 Variant，重生成 4 张 |
| "Set N 桌面好但移动不行，重做移动" | 保留 Set N 桌面，仅重生成 Set N 移动（同一 Variant，构图微调） |
